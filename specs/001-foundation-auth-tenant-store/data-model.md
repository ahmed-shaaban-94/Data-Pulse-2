# Phase 1 — Data Model

**Spec**: [spec.md §7](./spec.md) · **Plan**: [plan.md §3](./plan.md)
**Status**: Documentation only — no migrations generated yet.

This file translates the conceptual entities in spec §7 into a physical
PostgreSQL model. DDL snippets are illustrative documentation, not runnable
migrations. Actual migration files are produced in `/speckit-tasks` as
**explicit SQL files** authored with Drizzle Kit and committed to the repo
(per Constitution VIII).

---

## Conventions

- **PK**: `id UUID` (UUIDv7 generated by application; column `DEFAULT NULL`,
  app supplies it). Internal surrogate keys are not introduced in v1.
- **Tenant scoping**: every tenant-owned table carries `tenant_id UUID NOT NULL
  REFERENCES tenants(id) ON DELETE RESTRICT`.
- **Soft delete**: `deleted_at TIMESTAMPTZ NULL` on entities that support it
  (tenants, stores, users, memberships, invitations). Default queries filter
  `deleted_at IS NULL`.
- **Timestamps**: every table has `created_at TIMESTAMPTZ NOT NULL DEFAULT
  now()` and `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()` (trigger updates
  on row change).
- **Naming**: `snake_case`, plural table names, singular column names.
- **Money**: not relevant to this foundation; called out in plan PR risk
  list.
- **RLS**: every tenant-owned table enables RLS. Connection middleware sets
  `SET LOCAL app.current_tenant = :tenant_id` at the start of each
  transaction; policies compare against
  `current_setting('app.current_tenant')::uuid`.

## 1. `users`

**Purpose**: human identity. Tenant-agnostic.

| Column | Type | Notes |
|---|---|---|
| `id` | UUID | PK (UUIDv7). |
| `email` | TEXT | Citext-normalized, UNIQUE WHERE `deleted_at IS NULL`. |
| `email_verified_at` | TIMESTAMPTZ NULL | Set on verification. |
| `password_hash` | TEXT | argon2id PHC string. NULL for users authenticated only via SSO (future). |
| `display_name` | TEXT | Optional. |
| `is_platform_admin` | BOOLEAN NOT NULL DEFAULT false | Cross-tenant capability gate. |
| `created_at`, `updated_at`, `deleted_at` | TIMESTAMPTZ | |

**Constraints**:
- Partial unique index `users_email_active_uidx` on `(lower(email))` WHERE `deleted_at IS NULL`.
- CHECK `email <> ''`.

**RLS**: NOT applied (users are not tenant-scoped). Access is gated at the
application layer (only platform admins or the user themselves can read a
user record).

---

## 2. `tenants`

**Purpose**: a customer account / organization.

| Column | Type | Notes |
|---|---|---|
| `id` | UUID | PK. |
| `slug` | TEXT | Human-readable, URL-safe. UNIQUE WHERE `deleted_at IS NULL`. |
| `name` | TEXT NOT NULL | Display name. |
| `status` | TEXT NOT NULL DEFAULT `'active'` | Enum: `active`, `suspended`, `pending`. |
| `created_at`, `updated_at`, `deleted_at` | TIMESTAMPTZ | |

**Constraints**:
- Partial unique index on `lower(slug)` WHERE `deleted_at IS NULL`.
- CHECK `slug ~ '^[a-z0-9][a-z0-9_-]{1,62}[a-z0-9]$'`.
- CHECK `status IN ('active','suspended','pending')`.

**RLS**: enabled. Policy: a row is visible iff
`tenants.id = current_setting('app.current_tenant')::uuid`
OR the calling user is a platform admin (set by a separate boolean GUC
`app.is_platform_admin`).

---

## 3. `stores`

**Purpose**: a branch/location within a tenant.

| Column | Type | Notes |
|---|---|---|
| `id` | UUID | PK. |
| `tenant_id` | UUID NOT NULL | FK → `tenants(id)` ON DELETE RESTRICT. |
| `code` | TEXT | Tenant-unique store code (e.g., `BRANCH-01`). |
| `name` | TEXT NOT NULL | Display name. |
| `is_active` | BOOLEAN NOT NULL DEFAULT true | Distinct from soft-delete. |
| `created_at`, `updated_at`, `deleted_at` | TIMESTAMPTZ | |

**Constraints**:
- Partial unique index `stores_tenant_code_uidx` on `(tenant_id, lower(code))` WHERE `deleted_at IS NULL`.
- Index `stores_tenant_idx` on `(tenant_id)` for hot path lookups.

**RLS**: enabled. Policy: `tenant_id = current_setting('app.current_tenant')::uuid`
OR platform admin.

**Invariant I-1** is enforced by the FK + NOT NULL.
**Invariant I-3** is enforced by `store_access` (see §5).

---

## 4. `memberships`

**Purpose**: a user's relationship to a tenant. Carries role and store-access
policy. (Spec §7 entity: `Membership`.)

| Column | Type | Notes |
|---|---|---|
| `id` | UUID | PK. |
| `tenant_id` | UUID NOT NULL | FK → `tenants(id)`. |
| `user_id` | UUID NOT NULL | FK → `users(id)`. |
| `role_id` | UUID NOT NULL | FK → `roles(id)`. (See §6 — role catalog is per-tenant in v1.) |
| `store_access_kind` | TEXT NOT NULL | `'all'` or `'specific'`. |
| `revoked_at` | TIMESTAMPTZ NULL | Set when membership is revoked; row remains for audit. |
| `created_at`, `updated_at`, `deleted_at` | TIMESTAMPTZ | |

**Constraints**:
- UNIQUE `(tenant_id, user_id)` WHERE `deleted_at IS NULL` — enforces **Invariant I-2**.
- CHECK `store_access_kind IN ('all','specific')`.
- FK `(role_id) → roles(id)` with a CHECK at the app layer that
  `roles.tenant_id = memberships.tenant_id`. (PG can also do this with a
  composite FK if `roles` carries `(tenant_id, id)` as a unique key.)

**RLS**: enabled by `tenant_id`.

---

## 5. `store_access`

**Purpose**: when a membership's policy is `'specific'`, the explicit list of
allowed stores. (Spec §7 entity: `StoreAccess`.)

| Column | Type | Notes |
|---|---|---|
| `membership_id` | UUID NOT NULL | FK → `memberships(id)` ON DELETE CASCADE. |
| `store_id` | UUID NOT NULL | FK → `stores(id)`. |
| `tenant_id` | UUID NOT NULL | FK → `tenants(id)`. **Denormalized** for I-3 enforcement. |
| `created_at` | TIMESTAMPTZ NOT NULL DEFAULT now() | |

**Constraints**:
- PK `(membership_id, store_id)`.
- Composite FK `(tenant_id, membership_id) → memberships(tenant_id, id)`
  AND `(tenant_id, store_id) → stores(tenant_id, id)` to enforce
  **Invariant I-3** at the database level (both records must share a tenant).
- Trigger or app check: `memberships.store_access_kind = 'specific'` for any
  row in `store_access`. (Documented invariant; can be enforced via partial
  CHECK if PG supports it via a generated column.)

**RLS**: enabled by `tenant_id`.

---

## 6. `roles`

**Purpose**: a named bundle of capabilities. v1 is **predefined per tenant**
(seeded at tenant creation) per Q2 default C — predefined now, with the
forward-compat permission table ready.

| Column | Type | Notes |
|---|---|---|
| `id` | UUID | PK. |
| `tenant_id` | UUID NULL | NULL for platform-level roles (e.g., `platform_admin`). |
| `code` | TEXT NOT NULL | Stable code: `owner`, `tenant_admin`, `store_manager`, `store_staff`, `platform_admin`. |
| `name` | TEXT NOT NULL | Display name. |
| `is_built_in` | BOOLEAN NOT NULL DEFAULT true | Future custom roles flip this to false. |
| `created_at`, `updated_at` | TIMESTAMPTZ | |

**Constraints**:
- UNIQUE `(tenant_id, code)` (treats `tenant_id IS NULL` as platform scope).
- CHECK `code ~ '^[a-z][a-z0-9_]{1,40}$'`.
- Composite unique `(tenant_id, id)` to support the composite FK in §4.

**RLS**: enabled. Policy: row visible iff `tenant_id IS NULL` (platform) OR
`tenant_id = current_setting('app.current_tenant')::uuid`.

**Note**: For v1, the **role-to-permission mapping is in code**, not a join
table. The `permissions` and `role_permissions` tables below exist for
forward-compat and are populated empty until needed.

## 9. `sessions`

**Purpose**: dashboard session record. (Spec §7 entity: `Session`.) Server is
the source of truth; Redis caches.

| Column | Type | Notes |
|---|---|---|
| `id` | UUID | PK; matches the cookie value. |
| `user_id` | UUID NOT NULL | FK → `users(id)`. |
| `active_tenant_id` | UUID NULL | Set after context-switch. FK → `tenants(id)`. |
| `active_store_id` | UUID NULL | FK → `stores(id)`. |
| `issued_at` | TIMESTAMPTZ NOT NULL | |
| `last_seen_at` | TIMESTAMPTZ NOT NULL | Sliding-expiry watermark. |
| `absolute_expires_at` | TIMESTAMPTZ NOT NULL | Absolute cap (24h default). |
| `revoked_at` | TIMESTAMPTZ NULL | Server-initiated revocation. |
| `user_agent`, `ip_at_issue` | TEXT | Audit. |

**Constraints**:
- CHECK: if `active_store_id IS NOT NULL` then `active_tenant_id IS NOT NULL`.
- CHECK: if `active_store_id IS NOT NULL`, the store's `tenant_id =
  active_tenant_id` (enforced at app layer; can be added as a trigger).
  This enforces **Invariant I-4**.
- Index `sessions_user_active_idx` on `(user_id) WHERE revoked_at IS NULL`.
- Index `sessions_expires_idx` on `(absolute_expires_at) WHERE revoked_at IS NULL`.

**RLS**: NOT applied (sessions are user-scoped, not tenant-scoped).

## 10. `auth_tokens`

**Purpose**: opaque bearer tokens for API consumers and (future) POS devices.
Per research T-3.

| Column | Type | Notes |
|---|---|---|
| `id` | UUID | PK. |
| `token_hash` | BYTEA NOT NULL UNIQUE | SHA-256 of the bearer secret. |
| `tenant_id` | UUID NULL | NULL for non-tenant-scoped tokens (rare). |
| `user_id` | UUID NULL | Either a user or a device — not both. |
| `device_id` | UUID NULL | **Future** — column reserved; FK added when device entity ships. |
| `store_id` | UUID NULL | Optional store scope. |
| `scope` | TEXT NOT NULL | e.g., `'dashboard_api'`, `'pos'` (reserved). |
| `issued_at` | TIMESTAMPTZ NOT NULL | |
| `expires_at` | TIMESTAMPTZ NOT NULL | |
| `revoked_at` | TIMESTAMPTZ NULL | Instant revocation. |

**Constraints**:
- CHECK `(user_id IS NOT NULL)::int + (device_id IS NOT NULL)::int = 1`.
- Index `auth_tokens_user_active_idx` on `(user_id, expires_at) WHERE revoked_at IS NULL`.
- Index `auth_tokens_tenant_idx` on `(tenant_id)`.

**RLS**: enabled by `tenant_id` for tenant-scoped tokens; platform-scoped
tokens visible only to platform admins.

## 11. `invitations`

**Purpose**: pending offer to join a tenant. (Spec §7 entity: `Invitation`.)

| Column | Type | Notes |
|---|---|---|
| `id` | UUID | PK. |
| `tenant_id` | UUID NOT NULL | FK → `tenants(id)`. |
| `email` | TEXT NOT NULL | Citext-normalized; not necessarily an existing user. |
| `role_id` | UUID NOT NULL | FK → `roles(id)`. Role at acceptance time. |
| `store_access_kind` | TEXT NOT NULL | `'all'` or `'specific'`. |
| `invited_store_ids` | UUID[] NOT NULL DEFAULT '{}'::uuid[] | Used when kind=`'specific'`. |
| `invited_by_user_id` | UUID NOT NULL | FK → `users(id)`. |
| `token_hash` | BYTEA NOT NULL UNIQUE | Hash of the URL-safe accept token. |
| `status` | TEXT NOT NULL DEFAULT `'pending'` | `pending`, `accepted`, `expired`, `revoked`. |
| `expires_at` | TIMESTAMPTZ NOT NULL | Default: now() + 7 days. |
| `accepted_by_user_id` | UUID NULL | Set on accept. FK → `users(id)`. |
| `accepted_at` | TIMESTAMPTZ NULL | |
| `created_at`, `updated_at`, `deleted_at` | TIMESTAMPTZ | |

**Constraints**:
- CHECK `status IN ('pending','accepted','expired','revoked')`.
- CHECK `store_access_kind IN ('all','specific')`.
- Partial UNIQUE index on `(tenant_id, lower(email))` WHERE `status = 'pending'` AND `deleted_at IS NULL`.

**RLS**: enabled by `tenant_id`.

## 12. `audit_events`

**Purpose**: immutable record of governance/security actions. (Spec §7
entity: `AuditEvent`.)

| Column | Type | Notes |
|---|---|---|
| `id` | UUID | PK. |
| `occurred_at` | TIMESTAMPTZ NOT NULL DEFAULT now() | |
| `actor_user_id` | UUID NULL | FK → `users(id)`. NULL for unauthenticated events (e.g., failed sign-in attempts). |
| `actor_label` | TEXT NULL | E.g., the email used in a failed sign-in (no full credential). |
| `tenant_id` | UUID NULL | The tenant the action affected. |
| `store_id` | UUID NULL | Optional. |
| `action` | TEXT NOT NULL | e.g., `auth.signin.failed`, `tenant.user.invite`, `context.switch.tenant`. |
| `target_type` | TEXT NULL | e.g., `user`, `tenant`, `store`, `membership`. |
| `target_id` | UUID NULL | |
| `metadata` | JSONB NOT NULL DEFAULT `'{}'` | Bounded shape; PII-redacted. |
| `request_id` | UUID NULL | Trace correlation. |

**Constraints**:
- Indexed on `(tenant_id, occurred_at DESC)` for compliance exports
  (FR-AUDIT-2).
- Indexed on `(action, occurred_at DESC)`.
- No `UPDATE` allowed by application code; INSERT-only at the application
  layer. (Postgres revoke-update-permission can be added if desired.)

**RLS**: enabled by `tenant_id`. Platform-admin events with `tenant_id IS
NULL` (or referencing the target tenant) are queryable only by platform
admins.

| Column | Type | Notes |
|---|---|---|

| `store_id` | UUID NULL | |
| `client_id` | TEXT NOT NULL | E.g., device ID or API key fingerprint. |
| `key` | TEXT NOT NULL | Provided by client in `Idempotency-Key` header. |
| `request_hash` | BYTEA NOT NULL | Hash of normalized request body. |
| `response_status` | INT NOT NULL | |

## 14. RLS pattern (illustrative)

The illustrative SQL is documentation of the policy intent, not a runnable
migration:

```sql
-- ALL tenant-owned tables follow this pattern.
ALTER TABLE stores ENABLE ROW LEVEL SECURITY;

CREATE POLICY stores_tenant_isolation ON stores
  USING (
    tenant_id = current_setting('app.current_tenant', true)::uuid
    OR current_setting('app.is_platform_admin', true) = 'true'
  );
```

`current_setting(name, true)` returns NULL when the GUC is unset, which
ensures policies fail closed if middleware forgets to set the context.

## 15. Migration plan (prose)

Migrations are authored as **explicit SQL files** via Drizzle Kit and
committed under `packages/db/drizzle/` (numbered, e.g.,
`0000_initial.sql`, `0001_add_audit.sql`). A thin runner applies them in
order; each file has a paired rollback (`*.down.sql` or documented inverse)
reviewed for lock duration on large tables. Tool-agnostic: the SQL files
remain valid even if Drizzle is later replaced.

Migration order will be:

1. Create extension `citext` (for case-insensitive email/slug uniques).
   `pgcrypto` is **not** required — UUIDs are generated in the application
   layer (UUIDv7 per T-5).
2. Create `users`, `tenants` (no FKs cross-cutting yet).
3. Create `roles`, `permissions`, `role_permissions` (catalog tables).
4. Create `stores`, `memberships`, `store_access` (with composite FKs to
   enforce I-3).
5. Create `sessions`, `auth_tokens`, `invitations`, `audit_events`.
6. Create `idempotency_keys`.
7. Enable RLS on every tenant-owned table; create policies.
8. Seed platform admin role and the predefined per-tenant role catalog
   template (applied on tenant creation).
9. Triggers for `updated_at` maintenance.

Each migration includes a rollback plan and is reviewed for lock duration
on tables expected to be large (audit, sessions). Per Constitution VIII.

## 16. Mapping back to spec invariants

| Spec invariant | Enforced by |
|---|---|
| I-1 (`Store.tenant_id` non-null + valid) | NOT NULL + FK on `stores.tenant_id`. |
| I-2 (`(user, tenant)` membership unique) | Partial UNIQUE on `memberships(tenant_id, user_id)` WHERE deleted_at IS NULL. |
| I-3 (`StoreAccess` and parent `Membership` share tenant) | Composite FK `(tenant_id, membership_id)` + `(tenant_id, store_id)`. |
| I-4 (active store ⇒ active tenant matches store's tenant) | CHECK on `sessions` + app-layer trigger. |
| I-5 (soft-deleted entities not visible to non-platform-admin) | RLS policies + default `deleted_at IS NULL` filter in Drizzle tenant-scoped helpers. |
